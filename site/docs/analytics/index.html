Consent mode

Declared before GTM on every page

Every page declares consent defaults immediately before the GTM script tag. This ensures GA4 receives explicit analytics_storage: granted before any tags fire.

Consent defaults snippet (head, before GTM)

// Inserted on every page before the GTM script tag
window.dataLayer = window.dataLayer || [];
function gtag(){dataLayer.push(arguments);}
gtag('consent', 'default', {
  'analytics_storage':    'granted',
  'ad_storage':           'denied',
  'ad_user_data':         'denied',
  'ad_personalization':   'denied'
});

Why this matters: Without an explicit consent default, GA4 encodes an ambiguous _eu consent parameter on every collect hit. This can cause collect requests to return 503. The explicit grant resolves the ambiguity and ensures hits are accepted.

Tracked events

Custom events fired from page code + GTM enhanced measurement

install_command_copied Custom · Homepage

Fires when a user clicks the copy button on the install command block. Indicates install intent.

Properties
PropertyTypeExampleNotes

cta_clicked Custom · Homepage

Fires on any click to a /pilot, /demo, or .btn-primary link. Captures conversion-intent CTA clicks with full context.

gtag('event', 'cta_clicked', {
  cta_text: 'Request pilot access',   // button label, max 60 chars
  cta_href: '/pilot/',                // destination href
  cta_type: 'pilot'                   // 'pilot' | 'demo' | 'primary'
});
Properties
PropertyTypeValues
cta_textstringVisible button text, truncated to 60 chars
cta_hrefstringThe href attribute of the clicked element
cta_typestring"pilot" | "demo" | "primary"

Source: site/index.html — delegated click listener on document

pilot_form_submitted Custom · /pilot/

Fires when the Formspree pilot request form submits successfully (HTTP 200 from Formspree). This is the primary conversion event for mnemehq.com.

gtag('event', 'pilot_form_submitted');
// No additional properties — the submission itself is the signal.

Source: site/pilot/index.html — Formspree fetch success handler (form ID: meennlwj)

form_submit Enhanced measurement · GTM ★ Key event

Auto-captured by GTM enhanced measurement on any native HTML form submission detected on the page. On the pilot page, this fires alongside pilot_form_submitted because the form uses a fetch intercept that still allows native form detection. Marked as a key event in GA4.

Source: GTM enhanced measurement — no code change needed

cta_demo_click GTM tag · Site-wide ★ Key event

Fired by an existing GTM tag when a demo CTA is clicked. Predates the cta_clicked event; both coexist. Marked as a key event in GA4.

Source: GTM container — GTM-KL7FB67N tag configuration

Additional events automatically captured by GTM enhanced measurement: page_view, session_start, first_visit, scroll, scroll_depth, click, file_download, outbound_link_clicked, form_start, user_engagement, benchmark_section_viewed, demo_page_viewed, use_case_viewed, contact_method_clicked, community_engagement_click, insight_article_clicked.

How to add new events

Pattern used across the codebase

All custom events use gtag('event', ...) directly. The gtag function is defined in the consent defaults snippet on every page, so it is always available. No GTM tag or trigger configuration is needed.

Minimal event

gtag('event', 'your_event_name');

Event with properties

gtag('event', 'your_event_name', {
  property_one: 'value',
  property_two: 42
});

Event with defensive gtag guard (use when outside the consent snippet scope)

window.gtag = window.gtag || function(){
  (window.dataLayer = window.dataLayer || []).push(arguments);
};
gtag('event', 'your_event_name', { ... });

Naming convention: lowercase snake_case, object-action format. Examples: pilot_form_submitted, cta_clicked, demo_step_completed. Avoid generic names like button_clicked — include context in the event name or a property.

After deploying, wait for the event to appear in GA4 Admin → Events → Recent events (can take up to 24 hours), then star it as a key event if it represents a conversion or meaningful intent signal. Update this document.

Debugging

Verify events are firing and reaching GA4

Check dataLayer state in DevTools console

window.dataLayer
// Should show gtm.js, gtm.dom, gtm.load — plus any custom events fired

Verify collect requests in DevTools Network tab

// Filter by: google-analytics.com/g/collect
// Look for:  en=page_view, en=cta_clicked, etc. in the request URL
// Status:    200 or 204 = success.  503 = property/stream issue.

Fire a test event manually

gtag('event', 'test_event', { source: 'devtools' });
// Then check Network for a /g/collect request with en=test_event

For real-time event monitoring, use GA4 Admin → DebugView (requires adding ?gtm_debug=x to the page URL or enabling the Google Analytics Debugger browser extension).
